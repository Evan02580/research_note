## 一、引言：跨符号表示的核心问题

### 1.1 什么是"跨符号中间表示"？

跨符号中间表示（Cross-symbolic Intermediate Representation）探索的核心问题是：

> **不同符号系统（数字、符号、表情、公式、代码等）在LLM内部是否存在统一的表示机制？**

这一问题的价值在于：
- 如果存在统一表示，则可以实现跨符号的泛化推理
- 如果存在统一表示，则可以通过一种符号系统解释另一种
- 理解统一表示机制有助于提升模型的可解释性与可信度

### 1.2 本综述的定位

本文并非直接研究跨符号问题，但提供了**理解LLM内部表示机制的核心框架**，这些框架可直接应用于跨符号研究：

```
跨符号统一性探索
      ↓
┌──────────────────────────────────┐
│  本综述提供的分析工具               │
├──────────────────────────────────┤
│  • 概念对齐：符号→统一概念空间       │
│  • 表示分解：表示空间的几何结构       │
│  • 功能透明：计算路径的可检查性       │
│  • 稀疏诱导：专业子电路发现          │
│  • 显式模块化：符号系统的模块化激活    │
└───────────────────────────────────┘
      ↓
  跨符号中间表示机制
```

## 二、跨符号研究的核心启示

### 2.1 概念对齐：符号→语义的桥梁

论文提出的**概念对齐（Concept Alignment）**原则是跨符号研究的最直接关联。

#### 核心思想
概念对齐鼓励潜在变量直接对应**人类可理解的语义概念**，减少多语义性（polysemanticity）。这正是跨符号统一性的核心假设：**不同符号可以映射到相同的语义概念**。

#### 相关方法

| 方法 | 对跨符号研究的启示 |
|------|------------------|
| **标准CBMs** | 通过硬瓶颈强制符号必须通过概念层，为跨符号映射提供了结构化框架 |
| **混合CBMs** | 允许概念通道与残差通道并行，可解释跨符号的"表层形式"与"深层语义"的分离 |
| **概念嵌入模型(CEMs)** | 将概念表示为高维向量而非标量，为不同符号系统提供了统一的语义空间表示 |
| **Codebook Features** | 无监督向量量化方法，可用于发现不同符号的离散码本表示，可能揭示符号的原子化语义单元 |

#### 跨符号应用假设

```
符号系统A (如：数学公式)      符号系统B (如：代码)
     ↓                            ↓
   编码器 g_A(x)                 编码器 g_B(x)
     ↓                            ↓
   概念空间 C = {c₁, c₂, ..., cₙ}  (统一)
     ↓                            ↓
   解码器 f(c)                   解码器 f(c)
     ↓                            ↓
   输出（跨符号泛化）            输出（跨符号泛化）
```

如果这一假设成立，则：
- 可以用公式解释代码行为
- 可以用表情解释文本情感
- 可以用数字解释抽象概念

### 2.2 表示分解性：符号空间的正交结构

论文的**表示分解性（Representational Decomposability）**原则揭示了表示空间的几何结构。

#### 核心思想
将潜在空间分解为**正交子空间**，使得不同语义因子可独立操作而不互相干扰。

#### 跨符号含义

如果LLM的表示空间具有可分解的正交结构，那么：

| 符号系统 | 可能对应的子空间 | 特征 |
|----------|----------------|------|
| 数字 | 数值/量级维度 | 可进行数学运算 |
| 表情 | 情感/极性维度 | 可进行情感分析 |
| 代码 | 语法/语义维度 | 可进行程序理解 |
| 公式 | 结构/逻辑维度 | 可进行符号推理 |

#### 关键论文引用

**Backpack Language Models** (Hewitt et al., 2023)：
- 将每个词分解为**多个义项向量**（sense vectors）
- 注意力机制约束为非负权重组合
- 输出表示为义项向量的加权组合

这一框架可直接扩展为：
```
词 → 多个义项
符号系统A → 多个符号原子
符号系统B → 多个符号原子
↓
跨符号义项对齐
```

---

### 2.3 功能透明性：符号计算的可检查路径

**功能透明性（Functional Transparency）**原则使符号处理的计算路径变得可检查。

#### 核心思想
模型既在结构上明确（通过模块/层次设计）又在语义上有意义（通过可解释函数），模型行为类似"可读算法"而非黑箱。

#### 对跨符号研究的方法论价值

| 方法 | 跨符号应用场景 |
|------|--------------|
| **GAMs/GA2M** | 分析不同符号对输出的加性贡献，可用于归因分析 |
| **KANs** | 将函数表示为样条，可视化符号→输出的映射曲线 |
| **B-cos Networks** | 产生线性解释，忠实反映符号参与推理的证据 |

#### 假设验证框架

```
输入：混合符号序列 [公式, 代码, 表情, 数字]
      ↓
过程：可解释的中间计算路径
      ↓
输出：生成结果
      ↓
验证：追溯每个符号的贡献度
```

---

### 2.4 稀疏自编码器：符号特征的直接发现

论文多次提到**稀疏自编码器（SAEs）**作为跨符号表示分析的关键工具。

#### 核心机制
SAEs将多语义激活分解为**稀疏、可解释的特征**，无需人工标注即可发现表示中的语义原子。

#### 跨符号研究价值

```
传统方法：
  符号 → 隐藏表示 → 解释困难

SAE方法：
  符号 → 隐藏表示 → SAE解码 → 离散特征 → 可解释特征
                           ↓
                    发现"数字特征"、"情感特征"、"语法特征"
```

#### 关键发现引用

Huben et al. (2024) 使用SAEs在语言模型中发现**高度可解释的特征**：
- 发现了对应于特定语义概念的稀疏特征
- 不同类型的激活可以被分解为独立的、可命名的主题

这一方法可直接应用于：
- 发现跨符号共享的原子特征
- 分析不同符号系统的激活模式差异
- 识别符号间表示的统一性与差异性

### 2.5 显式模块化：符号系统的专家分工

**显式模块化（Explicit Modularization）**通过MoE架构揭示了符号处理的专业化机制。

#### MoE与跨符号的关联

论文指出，MoE架构中专家的专业化是**可预测和可干预的**：

> "路由决策遵循独特的层间模式，可以预测性地引导以改变模型行为。" (Bandarkar et al., 2025; Zheng et al., 2025)

#### 跨符号模块化假设

```
┌─────────────────────────────────────┐
│           符号路由器                 │
│  [公式] → 专家1 (结构/逻辑专家)      │
│  [代码] → 专家2 (语法/语义专家)       │
│  [表情] → 专家3 (情感/极性专家)       │
│  [数字] → 专家4 (数值/量级专家)       │
└─────────────────────────────────────┘
```

#### 支持证据

- **Lingual-SMoE** (Zhao et al., 2024)：语言引导的路由，不同专家处理不同语言特征
- **Apollo-MoE** (Zheng et al., 2025)：按语言家族组织专家
- 这表明模型确实可以按符号类型进行专业化分工

## 三、跨符号统一性的证据与挑战

### 3.1 支持统一性的证据

| 证据来源 | 内容 | 对跨符号的意义 |
|----------|------|---------------|
| **CoCoMix** (Tack et al., 2025) | 概念预测与混合，跨概念推理 | 不同符号可共享高层语义概念 |
| **Backpack** | 义项向量加权组合 | 符号的语义可以分解为原子单元 |
| **概念嵌入模型** | 跨概念语义相似性 | 不同符号可映射到相似的语义空间 |
| **混合CBMs** | 概念+残差双通道 | 表层符号差异可与深层语义分离 |

### 3.2 挑战与待解决问题

| 挑战 | 描述 | 研究方向 |
|------|------|----------|
| **多语义性问题** | 单个符号可能承载多重语义 | 需借鉴概念对齐中的去多义性方法 |
| **跨模态对齐成本** | 不同符号系统需要对齐到统一概念空间 | 混合CBM的对抗训练方法 |
| **符号原子化** | 如何确定符号的最小语义单元 | Codebook Features的无监督发现方法 |
| **可解释性vs表达力** | 强制统一表示可能损失表达能力 | 稀疏诱导与动态专业化的平衡 |

## 四、跨符号研究的分析框架

基于本综述，提出以下跨符号表示分析框架：

### 4.1 层次化分析模型

```
┌─────────────────────────────────────────────────────────────┐
│                      符号输入层                              │
│     [数字] [符号] [表情] [公式] [代码] [自然语言]           │
└─────────────────────────────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────────┐
│                   编码与概念映射层                           │
│   g₁(x)    g₂(x)    g₃(x)    g₄(x)    g₅(x)    gₙ(x)       │
│   (符号→概念的专用编码器)                                  │
└─────────────────────────────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────────┐
│                 统一概念空间 C                               │
│        c₁, c₂, ..., cₖ (跨符号共享的语义维度)               │
│   - 数值维度：数字、量级词                                   │
│   - 情感维度：表情、情感词                                   │
│   - 结构维度：公式、代码语法                                │
│   - 语义维度：自然语言理解                                   │
└─────────────────────────────────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────────┐
│                 解码与生成层                                 │
│   f(c) → 跨符号泛化输出                                      │
│   - 公式可以解释代码                                         │
│   - 表情可以辅助情感推理                                     │
│   - 数字可以验证抽象概念                                      │
└─────────────────────────────────────────────────────────────┘
```

### 4.2 研究方法对应

| 研究问题 | 对应方法 | 实施路径 |
|----------|----------|----------|
| 不同符号是否共享概念空间？ | CBM分析 | 训练跨符号CBM，观察概念瓶颈层的激活 |
| 符号的原子语义单元是什么？ | Codebook Features + SAE | 使用VQ发现离散码本，使用SAE分解激活 |
| 符号间的语义距离如何度量？ | 概念嵌入模型 | 将符号映射到统一向量空间，计算余弦距离 |
| 跨符号推理的计算路径是什么？ | 功能透明分析 | 使用GAM/KAN可视化符号→输出的映射 |
| 不同符号是否激活不同专家？ | MoE分析 | 分析专家路由模式与符号类型的对应关系 |

## 五、核心结论与研究建议

### 5.1 核心结论

1. **本综述的核心贡献**：提供了理解LLM内部表示机制的五大设计原则，这些原则可作为跨符号研究的方法论框架

2. **概念对齐是跨符号统一性的理论基础**：如果不同符号可以映射到统一的概念空间，则跨符号泛化和解释成为可能

3. **表示分解性揭示了符号空间的几何结构**：正交子空间的存在表明符号可能具有可独立操作的语义维度

4. **稀疏表示是发现跨符号原子的关键**：SAE和Codebook方法可用于发现不同符号共享的语义特征

5. **模块化提供了符号专业化的证据**：MoE架构中的专家专业化表明模型确实可以区分不同类型的符号输入

### 5.2 研究建议

| 方向 | 具体建议 |
|------|----------|
| **实证研究** | 使用SAE分析不同符号系统的激活模式，验证是否存在共享特征 |
| **架构设计** | 借鉴混合CBMs设计，尝试建立符号→概念的显式映射结构 |
| **评估方法** | 开发跨符号一致性评估指标，验证不同符号是否映射到相同的概念 |
| **应用场景** | 在代码生成、数学推理、情感分析等场景中测试跨符号泛化能力 |
